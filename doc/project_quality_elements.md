[프로젝트 II] 대상 시스템 품질 요소 추정 보고서

사용자가 원하는 기대치는 무엇인가?

- 성능: 파일 검색 결과가 1초 이내에 표시되어야 하며, 대용량 파일 업로드 시 네트워크 대역폭의 80% 이상을 활용하여 지연 시간을 최소화함.
- 가용성: 200명의 동시 접속자가 발생하더라도 시스템 다운타임 없이 99.9%이상의 가용성을 보장함.
- 기능 정확성: 시스템의 기능이 명세서대로 정확하게 동작하고 오류가 없어야 함
- 신뢰성: 사용 중 고장이 발생하지 않아야 하며 안정적인 서비스가 보장되어야 함
- 사용성: 사용자가 배우기 쉽고 편리해야 하며, 직관적인 UI/UX를 제공해야 함
- 낮은 비용: 사용자는 적은 비용으로 고품질의 서비스를 이용하기를 기대함.

개발자 혹은 유지보수 담당자가 원하는 기대치는 무엇인가?

- 가독성: 소스 코드를 이해하기 쉬워야 하며 표준 코딩 스타일을 준수해야 함.
- 유지보수성: 코드를 수정하거나 기능을 추가하기 쉬운 구조여야 하며 변경 영향이 국한되어야 함
- 검증 가능성: 특정 기능이 잘 작동하는지 독립적으로 테스트하고 자동화된 검증이 가능해야 함
- 문서화: 코드와 일관성 있는 문서가 제공되어 개발 및 관리 효율성을 높여야 함

개발 조직의 관리자가 원하는 기대치는 무엇인가?

- 비용 효율성: 적은 비용으로 소프트웨어가 개발되어 비즈니스 이득이 창출되어야 함.
- 재사용성: 기존 컴포넌트의 재사용을 통해 개발 기간을 단축하고 생산성을 높이기를 원함
- 가시성: 개발 단계의 진척도, 산출물 상태, 품질 정보를 체계적으로 관리하여 의사결정이 가능해야 함
- 적응성: 개발된 결과물이 다른 응용 환경에서도 유연하게 사용되기를 기대함
