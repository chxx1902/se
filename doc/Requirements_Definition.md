## 1. 서론

### 1.1 문서의 목적 및 범위

이 문서는 **에어 드라이브(Air Drive)** — 경량화된 팀 협업을 위한 분산 파일 관리 및 히스토리 추적 시스템 — 에서 각 요구사항이 무엇인지 조사하고 정의하는 문서이다. 이 문서는 기능적, 비기능적, 인터페이스에 요구되는 사항들을 정의한다.

---

### 1.2 대상 시스템 개요

#### 1.2.1 프로젝트 정의

본 시스템은 대규모 클라우드 서비스의 복잡함을 걷어내고, **팀 단위의 빠른 자료 공유에 최적화된 경량 웹 스토리지**를 제공한다.

사용자는 웹을 통해 파일을 업로드하고, 팀원 간의 실시간 댓글 피드백을 통해 문서의 수정 방향을 공유할 수 있다. 또한 변경된 부분만 관리하는 **버전 제어 기능**을 제공하여 팀의 데이터 자산을 안전하게 보호한다.

#### 1.2.2 주요 기능 설명

| ID | 기능명 | 설명 |
|----|--------|------|
| F01 | 드래그 앤 드롭 파일 관리 | 복잡한 설치 없이 브라우저에서 파일을 바로 올리고 폴더별로 분류하는 직관적 관리 기능 |
| F02 | 시간 제한부 공유 링크 생성 | 보안을 위해 특정 시간(예: 1시간, 24시간) 동안만 활성화되는 임시 다운로드 링크 발급 기능 |
| F03 | 파일별 협업 댓글 | 파일 자체에 댓글을 달아 수정 이력이나 팀원 간의 의견을 기록하고 보존하는 기능 |
| F04 | 최근 활동 로그 추적 | 누가 어떤 파일을 언제 수정하거나 다운로드했는지 실시간으로 보여주는 가시성 제공 기능 |
| F05 | 키워드 기반 고속 필터링 | 파일명, 확장자, 태그를 조합하여 수천 개의 파일 중 필요한 자료를 즉시 찾아내는 검색 기능 |

---

### 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 버전 제어 | 파일의 변경 이력을 관리하고, 이전 상태로 복원할 수 있는 기능 |
| 시간 제한부 링크 | 일정 시간이 지나면 자동으로 만료되는 임시 공유 URL |
| 활동 로그 | 파일의 업로드, 수정, 다운로드 등 사용자 행동 기록 |
| 드래그 앤 드롭 | 마우스로 파일을 끌어다 놓는 방식의 UI 인터랙션 |
| API | 응용 프로그램에서 사용할 수 있도록 운영체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 만든 인터페이스 |

## 2. 요구사항

### 2.1 기능적 요구사항

#### 2.1.1 사용자 인증 기능

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-001 | 사용자는 아이디, 비밀번호, 이메일 주소를 입력하여 회원가입을 할 수 있다. | 상 |
| FR-002 | 사용자는 아이디, 비밀번호를 입력하여 로그인한다. | 상 |
| FR-003 | 사용자는 자신이 속한 팀의 파일 목록을 확인할 수 있다. | 상 |

#### 2.1.2 파일 관리 기능 (F01)

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-004 | 사용자는 드래그 앤 드롭 방식으로 파일을 업로드할 수 있다. | 상 |
| FR-005 | 사용자는 파일을 폴더별로 분류하여 관리할 수 있다. | 상 |
| FR-006 | 사용자는 업로드된 파일을 삭제할 수 있다. | 중 |

#### 2.1.3 공유 링크 기능 (F02)

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-007 | 사용자는 파일에 대해 시간 제한이 있는 공유 링크를 생성할 수 있다. | 상 |
| FR-008 | 시스템은 설정된 시간(1시간, 24시간 등)이 경과하면 공유 링크를 자동으로 만료시킨다. | 상 |
| FR-009 | 링크 수신자는 로그인 없이 만료 전 파일을 다운로드할 수 있다. | 중 |

#### 2.1.4 협업 댓글 기능 (F03)

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-010 | 사용자는 파일에 댓글을 작성할 수 있다. | 상 |
| FR-011 | 사용자는 자신이 작성한 댓글을 수정하거나 삭제할 수 있다. | 중 |
| FR-012 | 시스템은 댓글 작성 시 작성자와 작성 시간을 함께 기록한다. | 상 |

#### 2.1.5 활동 로그 기능 (F04)

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-013 | 시스템은 파일의 업로드, 수정, 다운로드 이벤트를 실시간으로 기록한다. | 상 |
| FR-014 | 사용자는 팀 전체의 최근 활동 로그를 시간순으로 확인할 수 있다. | 상 |
| FR-015 | 사용자는 특정 파일의 활동 이력만 필터링하여 확인할 수 있다. | 중 |

#### 2.1.6 검색 및 필터 기능 (F05)

| 분류 | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-016 | 사용자는 파일명, 확장자, 태그를 키워드로 파일을 검색할 수 있다. | 상 |
| FR-017 | 시스템은 검색 결과를 실시간으로 필터링하여 표시한다. | 상 |
